Grouper

8/24/15

The group_by enumberable method is incredibly useful. This has become especially apparent to me after trying to complete the challenges at Dev Bootcamp during the last week without a complete understanding of how this and other enumerables function.

As with any enumerable, group_by is a method that gets passed to each member of a collection. Group_by returns a hash that is organized according to the block that it is passed. A common example I have seen is the following:

At first, I did not entirely understand what was happening in this example. I can see that in the {0} hash, those are the numbers divisibe by 3, but how did we group the other two hashes? Since we are using modulo, those keys are referring to the remainder when we divide the numbers in the array by three. [2,5,8] each return a remainder of 2, and [1,4,7] each return a remainder of 1.

A small variation of this example more clearly illustrates how group_by is applying a code block to each object in our collection, and is then grouping the members by the result. Look at how the previous example is changed by adding a conditional ("== 0") to the block:

As you can see, there are now two hash pairs as opposed to the three before. This is because adding the "== 0" to the end of our block creates a conditional where the only two possible results are true or false. Thus, our array is now grouped via true/false, with the contents of the {0} hashkey from the previous example occupying {true}, and all the others now simply being grouped in {false}.

When you want to break an array into ordered groups, the group_by enumerable is the way to go. The key for the hash group will always be the result of whatever block you are passing in the method. This allows you to properly organize any group of things you are given, so that you can operate on them with specificity to complete a given objective.
